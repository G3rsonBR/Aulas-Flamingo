Aprendendo sobre as Tags principais e suas caracteristicas

Sobre as Tags:

Tags são elementos do HTML, com elas iremos criar diversas caixas com diferentes funções. Uma tag é composta de:

  • < - Sinal de menor
  • Nome da Tag
  • > - Sinal de maior
Isso é a estrutura básica de uma Tag. Algumas tags só terão essa estrutura, que chamamos de "abertura", mas 80% (ou mais) das Tags precisam de Aberta e Fechamento. O fechamento se dá pela seguinte estrutura:
  • < - Sinal de menor
  • / - Barra invertida
  • Nome da Tag
  • > - Sinal de maior
Juntando tudo isso, a nossa tag seria escrita da seguinte forma: <nomeDaTag> </nomeDaTag>
Tags como essa, escritas acima, precisam de um conteúdo, ou seja, um texto interno. Um exemplo é a tag <p>, sua estrutura é:
<p> Conteúdo do nosso Parágrafo </p>
Já tags como <img> são só isso, não precisam de conteúdo, então não tem fechamento.

Tags de estruturação do HTML:

  • <!DOCTYPE>: Mostra que estamos usando a última versão do HTML (v5)
  • <html>: Fala para o navegador que estamos criando uma estrutura HTML
  • <head>: É a cabeça da nossa página, todas as configurações que não vemos
  • <meta>: Configurações da página HTML
  • <title>: É o título do nosso HTML, é o texto que fica na aba do browser
  • <body>: É o corpo da nossa página, o que o usuário irá ver

Sobre atributos e valores:

Atributos são como Caracteristicas de um Elemento HTML (tag). Imagine que o elemento é uma pessoa, e essa pessoa pode ter algumas caracteristicas, por exemplo:
<pessoa bonito="verdadeiro" genero="masculino" >

  • Pessoa é o elemento
  • Bonito e Genero são caracteristicas
  • Verdadeiro e Masculino são os valores dessas caracteisticas

Estruturação de Pastas:

Precisamos sempre manter uma boa organização em nosso projeto, isso ajuda para que identifiquemos o que estamos fazendo. Um exemplo:

  • Imagine que você está criando um novo projeto
  • Esse projeto terá diversas páginas, ou seja, vários arquivos HTML, com diversas categorias diferentes
Nessa situação, concorda que se todos esss arquivos estiverem soltos, ficará dificil separar por "categoria"?
Por isso criamos a esrtuturação de pastas. Uma boa estruturação é definida com uma boa legibilidade. Bater o olho e identificar.
Nesse projeto, temos a separação de arquivos html de Aulas e de arquivos de Desafios, sabendo essa informação você já sabe do que se trata. Então mantenha seu projeto com uma boa estrutura de pastas.

Imagens:

Agora falando de imagens, temos uma tag que nos ajuda a fazer esse serviço: <img>
Para usar essa tag, temos que saber de algumas coisas: <img src="" alt="" >

  • img - é o nome do elemento (tag)
  • src & alt - são os atributos (caracteristicas) do elemento
  • "" - é o valor do atributo
Quando se trata de uma imagem, temos 2 formas de trazer elas para nossa página:
  • <img src="https://i.pinimg.com/736x/93/dc/88/93dc88f67ac295788dbef804539b6017.jpg" alt="Logo do Jogo AC Unity" > - Dessa forma, apenas trazemos uma imagem por meio de um link, ela renderiza a seguinte imagem:

      Vantagens:

    • Não ocupa memoria do nosso sistema, afinal é uma imagem na web

      • Desvantagens:

    • Depende de internet para ser carregada
    • Se a imagem cair, ela não irá aparecer na nossa página

  • <img src="../assets/imgs/acLogo.jpg alt="Logo do Jogo AC Unity" > - Dessa forma, trazemos uma imagem dentro do nosso projeto, por meio do "caminho de diretório" onde a salvamos, ela renderiza a seguinte imagem:

      Vantagens:

    • Não depende da internet para carregar
    • A imagem nunca vai sumir, a menos que deletemos ela

      • Desvantagens:

    • Dependendo da resolução, a imagem se torna pesada, ocupando memória do nosso sistema
Cada um possui sua vantagem e desvantagem, vai de você escolher qual é o melhor para sua necessidade.
